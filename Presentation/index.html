Problem Statement

  • Fragmentation: Separate platforms for different media types (IMDb, Goodreads).
  • Complexity: Existing sites are often cluttered with ads and complex navigation.
  • Security: Lack of moderator-vetted registration leads to spam and unverified accounts.
  • Inconsistency: Users struggle to track their multi-media consumption in one place.

Objectives

  • To create a Unified Hub for reviewing diverse media categories.
  • To implement Role-Based Security (User vs. Pending Admin vs. Approved Admin).
  • To design an Interactive Rating System (1-5 Stars).
  • To build a Scalable Backend using Node.js and MySQL.
  • To provide a premium Responsive UX for all devices.

System Architecture

  • Frontend: React.js SPA (Single Page Application).
  • Backend: Node.js & Express API Server.
  • Database: MySQL Relational Storage.
  • Security: Bcrypt Password Hashing.

USER → REACT → EXPRESS → MySQL

Technologies Used

Frontend: React 19, Vite, Axios, CSS3 Transitions

Backend: Node.js 20, Express, MySQL2 Connector

Security: Bcrypt.js, Environment Variables

Tools: VS Code, MySQL Workbench, Git

Database Design

  • Users: ID, Name, Email, Hash_Password, Role, Status
  • Admin_Requests: Queue for manual vetting.
  • Media: Separate tables for Movies, Books, TV Series.
  • Reviews: Linked by Foreign Keys to Media and User IDs.

Implementation

Home Dashboard: Featuring premium card layouts and glassmorphism styling for movies and books.

Results & Analysis

  • Performance: Sub-second page loads due to Vite build optimization.
